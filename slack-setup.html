🎩 Slack Setup

How to connect Harold (OpenClaw) to a Slack workspace via a custom bot app.

✅

Currently active

Harold is connected to Slack via Socket Mode. DMs from U041V2TG0 are allowlisted and working.

1

Create a Slack App

  1. Go to api.slack.com/apps → Create New App → From scratch
  2. Name it (e.g. "Harold") and pick your workspace
  3. Under OAuth & Permissions, add these Bot Token Scopes:
app_mentions:read channels:history channels:read chat:write groups:history groups:read im:history im:read im:write mpim:history mpim:read mpim:write reactions:write users:read

4. Install the app to your workspace → copy the Bot User OAuth Token (xoxb-...)

2

Enable Socket Mode

  1. Go to Settings → Socket Mode → toggle it on
  2. Generate an App-Level Token with scope connections:write
  3. Copy the token (xapp-...) — you'll need it in step 4
3

Enable Event Subscriptions

  1. Go to Event Subscriptions → toggle on
  2. Under Subscribe to bot events, add:
app_mention message.channels message.groups message.im message.mpim
4

Configure OpenClaw

Run the configure wizard and follow the prompts:

openclaw configure

Select Slack, then paste in your Bot Token and App Token when prompted.

5

DM Policy — Allowlist Setup

By default, DMs are locked to an allowlist. To receive DMs from yourself, add your Slack User ID.

Find your user ID: click your name in Slack → View profile → ⋯ More → Copy member ID.

The relevant config section:

"slack": {
  "mode": "socket",
  "dmPolicy": "allowlist",
  "allowFrom": ["U041V2TG0"],
  ...
}
Tip: To open DMs to everyone in your workspace, set dmPolicy to "open".
6

Channel Access (Optional)

To have Harold respond in a channel when mentioned, add it to the config:

"CHANNEL_ID": { "allow": true, "requireMention": true }

The channel ID can be found by right-clicking the channel in Slack → View channel details → scroll to the bottom.

Setting Value What it means
